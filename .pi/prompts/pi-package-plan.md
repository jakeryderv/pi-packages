# Pi Package Plan

Plan this Pi package task before implementation:

$ARGUMENTS

Use `AGENTS.md`, package docs, and the `pi-package-authoring` skill. Produce a
concise implementation plan that covers:

- files/modules to change,
- package manifest or dependency changes,
- Pi extension lifecycle concerns,
- data/storage/security concerns,
- tests or checks to add/update,
- preflight commands to run.

Do not implement until the plan is clear or any important ambiguity is resolved.
